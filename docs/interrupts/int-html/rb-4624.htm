DPMI 0.9+ - GET VENDOR-SPECIFIC API ENTRY POINT

AX = 168Ah
DS:(E)SI = selector:offset of ASCIZ vendor name (see #02719)


Return:
AL = status
00h successful
ES:(E)DI -> extended API entry point
8Ah unsuccessful

Notes: The vendor name is used to determine which entry point to return; it is case-sensitive. Available in protected mode only. 32-bit applications use ESI and EDI, 16-bit applications use SI and DI. This call is present but not documented for DPMI 0.9. The Borland C++ 3.1 DPMILOAD does not handle requests for entry points other than the MS-DOS one gracefully, producing an unhandled exception report; this has been fixed in the Borland Pascal 7 version

See Also: AX=1687h - INT 31/AX=0A00h - INT 31/AH=57h
